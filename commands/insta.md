# /insta

현재 세션의 대화 내용을 요약하여 **인스타그램에 바로 올릴 수 있는 콘텐츠**로 변환합니다.

## Instructions

### Phase 1: 세션 요약

현재 세션에서 이 `/insta` 명령어 호출 직전까지의 대화 내용을 분석하여 다음을 추출하라:

1. **핵심 주제** — 이 세션에서 다룬 메인 토픽 (1-2개)
2. **핵심 인사이트** — 세션에서 나온 가장 가치 있는 발견/결론 (3-5개)
3. **타겟 독자** — 이 내용에 관심 가질 만한 사람 (개발자, PM, 창업가, 일반인 등)
4. **한 줄 요약** — 세션 전체를 한 문장으로

### Phase 2: 포맷 선택

`$ARGUMENTS`를 확인하여 출력 포맷을 결정하라:

- `$ARGUMENTS`에 "릴스" 또는 "reels"가 포함되면 → **릴스 스크립트만** 생성
- `$ARGUMENTS`에 "카드" 또는 "card"가 포함되면 → **카드뉴스만** 생성
- `$ARGUMENTS`에 "둘다" 또는 "both"가 포함되면 → **릴스 + 카드뉴스 모두** 생성
- `$ARGUMENTS`가 비어 있거나 위에 해당하지 않으면 → 사용자에게 AskUserQuestion으로 질문:
  - "어떤 포맷으로 만들까요?"
  - 선택지: "릴스 스크립트", "카드뉴스", "둘 다"

`$ARGUMENTS`에 포맷 외 추가 지시사항이 있으면 Phase 3에서 반영하라. (예: "/insta 릴스 AI 도구 소개 느낌으로" → 릴스 스크립트 + "AI 도구 소개" 톤 반영)

### Phase 3: 콘텐츠 생성

아래 톤 가이드와 템플릿 구조를 따라 콘텐츠를 생성하라.

#### 톤 & 스타일 가이드

- **언어**: 한국어 기본
- **말투**: 캐주얼 해요체 ("~했어요", "~인데요", "~거든요")
- **이모지**: 적극 활용. 문장마다가 아닌, 포인트에 배치
- **문장 길이**: 짧고 임팩트 있게. 한 문장 15자 이내 권장
- **줄바꿈**: 많이. 한 줄에 하나의 메시지
- **전문 용어**: 쉬운 말로 풀어쓰기 (필요시 괄호 설명)
- **톤**: "나도 이거 몰랐는데 알게 됐어요" 느낌의 공유형 톤
- **금지**: 딱딱한 보고서체, 과도한 영어, "~하겠습니다" 같은 격식체

### Phase 4: 최종 출력

생성된 콘텐츠를 아래 마크다운 포맷으로 출력하라. 사용자가 그대로 복사해서 쓸 수 있어야 한다.

## 릴스 스크립트 구조

릴스를 선택했을 때 아래 구조를 따른다.

```
## 🎬 릴스 스크립트

### 📌 컨셉
> {한 줄 컨셉 설명}

### 🎯 타겟
{누구를 위한 콘텐츠인지}

---

### 🎞️ 장면 구성

**[장면 1 — 훅 (0-3초)]**
🎤 자막: "{관심을 끄는 첫 문장}"
📸 화면: {화면 설명}

**[장면 2 (3-8초)]**
🎤 자막: "{핵심 메시지 1}"
📸 화면: {화면 설명}

**[장면 3 (8-15초)]**
🎤 자막: "{핵심 메시지 2}"
📸 화면: {화면 설명}

... (총 5-8장면, 30초-1분)

**[마지막 장면 — CTA]**
🎤 자막: "{행동 유도 문구}"
📸 화면: {화면 설명}

---

### 🎵 BGM 분위기
{추천 BGM 분위기 — 예: "잔잔한 lo-fi", "빠른 테크 느낌", "트렌디한 팝"}

### #️⃣ 해시태그
{해시태그 10-15개, 공백으로 구분}

### 📝 캡션
{인스타 게시글 캡션 — 3-5줄 + CTA}
```

## 출력 예시

### 릴스 예시 — "Claude Code로 블로그 MVP 만든 세션"

```markdown
## 🎬 릴스 스크립트

### 📌 컨셉
> AI한테 "블로그 만들어줘" 했더니 진짜 만들어줌

### 🎯 타겟
코딩에 관심 있는 비개발자, AI 도구 얼리어답터

---

### 🎞️ 장면 구성

**[장면 1 — 훅 (0-3초)]**
🎤 자막: "명령어 하나로 블로그가 만들어진다면?"
📸 화면: 터미널에 /autopilot 치는 모습

**[장면 2 (3-8초)]**
🎤 자막: "Claude Code한테 블로그 MVP 만들어달라고 했어요"
📸 화면: Claude가 계획 세우는 화면 빠르게 스크롤

**[장면 3 (8-15초)]**
🎤 자막: "계획 → 실행 → 검증까지 알아서 해요"
📸 화면: Plan → Work → Review 흐름 도식

**[장면 4 (15-22초)]**
🎤 자막: "검증 실패하면? 자동으로 고치고 다시 검증"
📸 화면: FAIL → 수정 → PASS 되는 과정

**[장면 5 (22-27초)]**
🎤 자막: "결과물 퀄리티? 솔직히 놀랐어요"
📸 화면: 완성된 블로그 사이트 보여주기

**[장면 6 — CTA (27-30초)]**
🎤 자막: "이런 거 더 보고 싶으면 팔로우!"
📸 화면: 계정 프로필 + 팔로우 버튼

---

### 🎵 BGM 분위기
빠르고 트렌디한 테크 느낌 (lo-fi beats 또는 upbeat electronic)

### #️⃣ 해시태그
#ClaudeCode #AI코딩 #개발자일상 #노코드 #AI도구 #프로그래밍 #테크 #사이드프로젝트 #블로그만들기 #자동화 #개발일지 #AItools #코딩초보 #IT트렌드

### 📝 캡션
명령어 하나로 블로그 만들기 도전했어요 🚀

Claude Code의 /autopilot 기능인데요,
계획부터 코딩, 검증까지 다 알아서 해줘요.

진짜 되나 싶었는데... 됩니다 😳

AI 코딩 도구 리뷰 더 보고 싶으면 팔로우 해주세요!
저장해두면 나중에 써먹을 수 있어요 📌
```

### 카드뉴스 예시 — "1on1 미팅 조언 세션"

```markdown
## 📱 카드뉴스

### 📌 컨셉
> COO와 1on1에서 받은 커리어 조언 정리

### 🎯 타겟
스타트업 PM, 주니어 리더, 커리어 고민하는 직장인

---

### 슬라이드 구성

**[1/6 — 표지]**
> 🔥 COO가 알려준 "살아남는 PM"의 3가지 습관
> 스타트업에서 성장하고 싶다면 이것만 기억하세요

**[2/6]**
**습관 1: 숫자로 말하기**
• "느낌" 대신 데이터로 설득해요
• DAU, 전환율, 리텐션 — 항상 숫자를 준비
• 정확하지 않아도 괜찮아요. 추정이라도 있는 게 낫다

**[3/6]**
**습관 2: 실패를 빠르게 공유하기**
• 안 좋은 소식일수록 빨리 올려요
• "이런 문제 있는데, 이렇게 해볼 생각입니다"
• 문제 + 해결안 세트로 가져가면 신뢰가 쌓여요

**[4/6]**
**습관 3: 오너십 범위를 넓히기**
• 내 담당 아닌 것도 챙기는 사람이 결국 올라가요
• "그건 제 일 아닌데요"는 성장의 적
• 작은 것부터 — 옆 팀 이슈에 코멘트 한 줄 다는 것부터

**[5/6]**
**보너스: 1on1을 잘 쓰는 법**
• 보고 시간이 아니라 코칭 시간으로 활용
• "요즘 고민" → "조언 요청" → "피드백 확인"
• 아젠다는 내가 주도적으로 세우기

**[6/6 — CTA]**
> 💡 숫자, 빠른 공유, 넓은 오너십 — 이 3가지만 기억하세요
> 👉 도움이 됐다면 저장 + 팔로우!
> 📌 스타트업 PM의 성장 기록을 공유하고 있어요

---

### #️⃣ 해시태그
#PM #프로덕트매니저 #스타트업 #커리어성장 #1on1 #리더십 #직장인성장 #자기계발 #일잘러 #워킹맘 #스타트업일상 #커리어조언 #성장마인드셋

### 📝 캡션
COO와 1on1 하면서 받은 조언이 너무 좋아서 정리했어요 📝

특히 "숫자로 말하기"가 진짜 게임체인저였는데요,
발표할 때 체감 온도가 완전히 달라지더라고요.

여러분은 1on1에서 어떤 이야기 하세요?
댓글로 알려주세요! 👇

#PM커리어 #스타트업성장
```

## Usage

```
/insta                    → 포맷 선택 질문 후 생성
/insta 릴스               → 릴스 스크립트만 생성
/insta reels              → 릴스 스크립트만 생성
/insta 카드뉴스           → 카드뉴스만 생성
/insta card               → 카드뉴스만 생성
/insta 둘다               → 릴스 + 카드뉴스 모두 생성
/insta both               → 릴스 + 카드뉴스 모두 생성
/insta 릴스 AI 도구 느낌으로  → 릴스 + 추가 톤 지시 반영
```
