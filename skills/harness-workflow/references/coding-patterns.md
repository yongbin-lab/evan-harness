# 코딩 프로젝트 패턴 레퍼런스

## 지원하는 코딩 태스크 유형

### 1. MVP 구현 (새 프로젝트)
**Plan 단계 포커스:**
- 기술 스택 선정 (프로젝트 요구사항 기반)
- 핵심 기능 목록 (MVP 범위 한정)
- 디렉토리 구조
- 데이터 모델

**Criteria 예시:**
- 핵심 기능이 모두 동작하는가
- 개발 서버가 에러 없이 실행되는가
- 기본 라우팅/네비게이션이 작동하는가
- 코드에 하드코딩된 시크릿이 없는가

### 2. 기능 추가 (기존 프로젝트)
**Plan 단계 포커스:**
- 기존 코드베이스 분석 (패턴, 컨벤션)
- 영향 범위 (어떤 파일이 변경되는가)
- 기존 테스트와의 호환성
- 롤백 계획

**Criteria 예시:**
- 새 기능이 의도대로 동작하는가
- 기존 테스트가 모두 통과하는가
- 새 기능에 대한 테스트가 추가되었는가
- 기존 코드 패턴/컨벤션을 따르는가

### 3. 버그 수정
**Plan 단계 포커스:**
- 재현 단계
- 근본 원인 분석
- 수정 방향 (최소 변경)
- 회귀 테스트 계획

**Criteria 예시:**
- 버그가 재현되지 않는가
- 근본 원인이 해결되었는가 (증상만 가린 게 아닌가)
- 회귀 테스트가 추가되었는가
- 관련 없는 코드가 변경되지 않았는가

### 4. 리팩토링
**Plan 단계 포커스:**
- 리팩토링 목표 (성능? 가독성? 유지보수성?)
- Before/After 비교 포인트
- 단계별 마이그레이션 계획
- 기능 동등성 검증 방법

**Criteria 예시:**
- 모든 기존 테스트가 통과하는가
- 리팩토링 목표 메트릭이 개선되었는가
- 외부 API/인터페이스가 변경되지 않았는가
- 코드 라인 수가 감소했는가 (해당되는 경우)

## 공통 검증 체크리스트

- [ ] `npm test` / `pytest` 등 테스트 스위트 통과
- [ ] 빌드 성공 (`npm run build` 등)
- [ ] 린터 경고 없음
- [ ] .env / 시크릿 파일이 커밋에 포함되지 않음
- [ ] README 업데이트 (필요한 경우)
