# PM 업무 패턴 레퍼런스

## 지원하는 PM 태스크 유형

### 1. Epic/PRD 작성
**Plan 단계 포커스:**
- 유저 저니 (최소 3개)
- 기능 요구사항 (SMART: Specific, Measurable, Achievable, Relevant, Time-bound)
- 성공 메트릭 (정량적 KPI)
- 기술 제약사항

**Criteria 예시:**
- 유저 저니가 3개 이상 정의되었는가
- 각 FR이 SMART 기준을 충족하는가
- 성공 메트릭에 구체적 수치 목표가 있는가
- 엣지 케이스가 최소 3개 식별되었는가

### 2. 데이터 분석
**Plan 단계 포커스:**
- 분석 목적 (어떤 의사결정을 지원하는가)
- 데이터 소스 식별
- 분석 방법론
- 기대 산출물 형태

**Criteria 예시:**
- 분석 결과가 의사결정에 직접 연결되는가
- 데이터 소스가 명시되었는가
- 시각화가 포함되었는가
- 한계점/주의사항이 명시되었는가

### 3. 리서치
**Plan 단계 포커스:**
- 리서치 질문 (명확한 질문 3-5개)
- 조사 범위 및 소스
- 비교 프레임워크
- 산출물 형태

**Criteria 예시:**
- 리서치 질문에 대한 답변이 모두 포함되었는가
- 소스가 3개 이상 인용되었는가
- 비교 분석이 구조화되었는가
- Actionable 인사이트가 도출되었는가

### 4. 전략/플레이북 작성
**Plan 단계 포커스:**
- 현재 상황 분석
- 목표 상태 정의
- 액션 아이템 도출
- 타임라인

**Criteria 예시:**
- 현재-목표 Gap이 명확히 정의되었는가
- 액션 아이템에 담당자/기한이 있는가
- 리스크 및 대안이 포함되었는가
- 성공 기준이 정량적인가
